## 🎯 设计理念

中国大学生在撰写课程设计报告、商业计划书、实验报告、实习报告等文档时，**排版规范**往往是比内容本身更耗时、更容易出错的环节。大多数同学并非不懂内容，而是：

- ❌ 不知道封面、目录、正文的国标排版要求
- ❌ 手动调整字体、字号、行距浪费大量时间
- ❌ 插入图表后忘记编号或图注格式不统一
- ❌ 参考文献格式不符合 GB/T 7714 规范
- ❌ 三线表做成全边框表，被老师批评

**本 Skill 的目标是：让 AI 代理在生成文档的同时，自动完成全部排版工作，输出即交付——拿来就能交作业。**

## ✨ 功能特性

### 📋 完整样式系统
- **15 种自定义段落样式**（CoverTitle / Heading1_Course / BodyText_Course / Caption_Course …）
- 中英文双字体自动匹配（宋体+Times New Roman / 黑体+Arial）
- 所有样式符合 GB/T 7713.2-2022 国标要求

### 📐 三段式分节结构
封面（无页码） → 目录（罗马数字页码） → 正文（阿拉伯数字页码从1开始）

### 📊 智能图表引擎 (ChartEngine)
- **7 种 matplotlib 图表模板**：折线图 / 柱状图 / 饼图 / 雷达图 / 横向柱状图 / 分组柱状图 / 多系列折线图
- **自动中文字体配置**：动态查找 SimHei / YaHei / Heiti 等中文字体，杜绝图片中文乱码
- **智能配图**：按文档类型自动匹配 ≥8 张图表配置
- 图表类型 ≥4 种，同类型不连续超过 2 次

### 📑 标准三线表
- 严格三线表（顶线粗 + 栏目线细 + 底线粗，无竖线，数据行间无横线）
- 通过 `tcBorders` 单元格边框实现栏目线，符合 CY/T 170 规范
- 表题在上方、注释在下方

### 📝 正文规范
- 每段 **150-350 字**，禁止列表替代段落
- 每章字数分配 ±40%（含 5 类文档的字数分配表）
- 内置 `check_document_length()` 字数检查函数
- 章节过渡段落、图表数据解读规则

### 🔗 自动目录
- TOC 字段码，大纲级别确保导航窗格可识别
- 目录页码右对齐，点线前导符

### 🌐 英文摘要
- 所有文档自动生成英文 Abstract + Keywords
- 正文 >6000 字时摘要 ≥150 词

### 📚 参考文献
- GB/T 7714-2015 格式（期刊 / 专著 / 学位论文 / 报告 / 标准）
- 支持出版地字段
- 真实性验证规则

## 🚀 快速开始

### 前置条件

```bash
pip install python-docx matplotlib numpy
```

### 使用方法

1. **调用 Skill**：AI 代理通过 `/course-design-docx` 触发
2. **指定文档类型**：商业计划书 / 实验报告 / 软件课程设计 / 实习报告 / 毕业论文
3. **提供主题信息**：例如"低空经济"、"电商平台用户增长分析"
4. **等待生成**：AI 代理按完整工作流自动生成 .docx 文档

### 示例命令

```
/course-design-docx 请帮我生成一份「低空经济商业计划书」
```

## 🧪 质量检查清单

生成文档后，Skill 自动对照以下清单检查：

- ✅ 自定义样式完整（无 Normal 样式滥用）
- ✅ 三节页码控制（封面无/目录罗马/正文阿拉伯）
- ✅ 图表 ≥8 张，类型 ≥4 种
- ✅ 三线表（无竖线/无隔行底纹）
- ✅ 表题在上、注释在下
- ✅ 图注格式含数据来源
- ✅ 每段 150-350 字
- ✅ 章节字数均衡（±40%）
- ✅ 英文摘要完整
- ✅ 参考文献 GB/T 7714-2015
- ✅ 图片中文无乱码（matplotlib 字体配置）

## 📖 文档规范依据

| 标准编号 | 名称 | 适用范围 |
|---------|------|---------|
| GB/T 7713.1-2006 | 学位论文编写规则 | 页边距、封面结构 |
| GB/T 7713.2-2022 | 学术论文编写规则 | 章节编号、标题格式 |
| GB/T 7714-2015 | 参考文献著录规则 | 参考文献格式 |
| CY/T 170 | 学术出版规范 表格 | 三线表格式 |
